# שיתוף אוטומטי של חומרים ב-Google Drive

המערכת יכולה לשתף ולבטל שיתוף של הקלטות סשנים וחומרי קורסים **אוטומטית**,
בלי שאף אחת תיגע בדרייב. עד שמגדירים את זה — הכול עובד כמו קודם, עם התור
הידני ב**ניהול ← תור שיתופים**.

## מה קורה אוטומטית אחרי ההגדרה

| מתי | מה המערכת עושה |
|---|---|
| חברה משלמת ומצטרפת | משתפת איתה את **כל** הקלטות הסשנים |
| אדמין מאשרת חברה (סטטוס → פעילה) | אותו דבר |
| מוסיפים סשן חדש | משתפת אותו עם **כל החברות הפעילות** (לא רק עתידיות) |
| מוסיפים קישור לקורס/סשן קיים | משתפת אותו עם כל מי שכבר יש לה גישה לאותו קורס/סשן |
| חברה פותחת קורס | משתפת איתה את חומרי הקורס |
| חברה מחזירה קורס | **מבטלת** את השיתוף של אותו קורס |
| מנוי מסתיים / אדמין משהה או דוחה חברה | **מבטלת את כל** השיתופים שלה |

כל פעולה שנכשלת נשארת בתור הידני — שום דבר לא הולך לאיבוד.

## הגדרה חד-פעמית (10 דקות)

### 1. פרויקט וחשבון שירות ב-Google Cloud

1. נכנסים ל-<https://console.cloud.google.com> → **New Project** (שם: `open-code`).
2. תפריט → **APIs & Services → Library** → מחפשים **Google Drive API** → **Enable**.
3. תפריט → **APIs & Services → Credentials** → **Create Credentials → Service account**.
   - שם: `open-code-drive`. אין צורך בהרשאות בשלב הבא — פשוט **Done**.
4. נכנסים לחשבון השירות שנוצר → לשונית **Keys** → **Add key → Create new key → JSON**.
   הקובץ יורד למחשב. בתוכו שני שדות שנצטרך: `client_email` ו-`private_key`.

### 2. לתת לחשבון השירות גישה לחומרים

זה החלק הקריטי — בלעדיו Google יסרב לשתף.

**האפשרות המומלצת: כונן שיתופי (Shared Drive)**

1. בדרייב → **כוננים שיתופיים → כונן שיתופי חדש** (למשל "קוד פתוח — תכנים").
2. מעבירים לתוכו את כל תיקיות הקורסים וההקלטות.
3. לוחצים על הכונן → **ניהול חברים** → מוסיפים את כתובת ה-`client_email`
   של חשבון השירות בתור **מנהל תוכן (Content manager)**.

**אפשרות חלופה:** אם החומרים נשארים ב"הדרייב שלי", משתפים את תיקיית העל
עם ה-`client_email` בהרשאת **עורך**, ומוודאים שבהגדרות השיתוף מסומן
שעורכים רשאים לשנות הרשאות ולשתף.

### 3. משתני סביבה ב-Vercel

**Settings → Environment Variables** (Production), ואז **Redeploy**:

| שם | ערך |
|---|---|
| `GOOGLE_SERVICE_ACCOUNT_EMAIL` | ה-`client_email` מקובץ ה-JSON |
| `GOOGLE_PRIVATE_KEY` | ה-`private_key` מקובץ ה-JSON — **כולל** `-----BEGIN PRIVATE KEY-----` ו-`-----END PRIVATE KEY-----`, עם ה-`\n` כמו שהם בקובץ |

> אם עורכים את המפתח ביד — לא להחליף את רצפי `\n` בשורות חדשות. הקוד מטפל בזה.

### 4. בדיקה

נכנסים ל**ניהול ← תור שיתופים**. בראש המסך תופיע רצועה ירוקה
**"שיתוף אוטומטי פעיל"**. אם היא כתומה — יש שם גם הסבר מה חסר.

בדיקה אמיתית: מוסיפים סשן ניסיון עם קישור דרייב, ואז מאשרים חברת בדיקה —
היא אמורה לקבל גישה בלי שאף אחת תיגע בדרייב.

## מגבלות שכדאי להכיר

- **הכתובת שאיתה משתפים היא כתובת ההתחברות של החברה.** אם היא נרשמה עם
  כתובת שאינה חשבון Google, גוגל ישלח לה הזמנה למייל והיא תצטרך לפתוח
  חשבון Google על אותה כתובת (או להיכנס עם אחד קיים) כדי לצפות.
- **ביטול שיתוף מסיר את ההרשאה, לא עותקים.** אם מישהי הורידה קובץ למחשב
  שלה לפני שעזבה — זה כמובן נשאר אצלה.
- **הקישורים חייבים להיות קישורי Drive** (קובץ, תיקייה, Docs/Sheets/Slides).
  קישור מסוג אחר פשוט יישאר בתור הידני.
